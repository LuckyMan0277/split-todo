- ✅ Text Primary (#1f2937) on white (#ffffff) - 대비율 약 15.5:1 (통과)

**주의 필요**:
- ⚠️ Text Secondary (#6b7280) on white (#ffffff) - 대비율 약 4.6:1 (AA 통과하지만 여유 없음)
- ⚠️ Text Secondary (#6b7280) on Background (#f9fafb) - 대비율 약 4.5:1 (경계선)
- ⚠️ Success Green (#10b981) on white - 대비율 약 2.8:1 (실패)
- ⚠️ Danger Red (#ef4444) on white - 대비율 약 3.9:1 (실패)

### 타이포그래피

#### 현재 상태

**폰트 크기**:
- Title: 28px
- Large: 20-24px
- Medium: 16-18px
- Small: 14px

**폰트 무게**:
- Bold: 700
- Semibold: 600
- Regular: 400

**줄 간격**: 명시되지 않음 (⚠️)

#### 평가

**장점**:
- ✅ 폰트 크기 위계가 명확하고 충분한 차이
- ✅ 본문 텍스트 16px 이상으로 가독성 확보
- ✅ 폰트 무게 단계가 적절 (3단계)

**개선 필요**:
- ⚠️ 줄 간격(line height) 미정의
- ⚠️ 글자 간격(letter spacing) 미정의
- ⚠️ 최대 텍스트 너비 미정의 (가독성 위해 중요)

#### 개선 제안

**High (출시 전 필수)**:

1. **줄 간격 정의**
   ```typescript
   typography: {
     h1: { size: 28, weight: '700', lineHeight: 36 },  // 1.29배
     h2: { size: 24, weight: '600', lineHeight: 32 },  // 1.33배
     h3: { size: 20, weight: '600', lineHeight: 28 },  // 1.4배
     body: { size: 16, weight: '400', lineHeight: 24 }, // 1.5배
     caption: { size: 14, weight: '400', lineHeight: 20 }, // 1.43배
   }
   ```
   - 이유: 줄 간격이 없으면 텍스트가 답답하고 가독성 저하

2. **긴 텍스트 처리 규칙**
   - Task 제목: 최대 2줄, 초과 시 말줄임표(...)
   - Checklist 항목: 최대 3줄, 초과 시 말줄임표
   - 이유: 레이아웃 일관성 및 UI 깨짐 방지

**Medium (개선 권장)**:

3. **글자 간격 최적화**
   - Title (28px): letter-spacing -0.5px (타이트하게)
   - Body (16px): letter-spacing 0px (기본)
   - Caption (14px): letter-spacing 0.25px (약간 여유)

4. **다국어 지원 고려**
   - 현재 한글 기준 설계로 보임
   - 영어/일본어 등 폰트 크기 조정 필요 시 고려

## 📱 화면별 피드백

### TaskListScreen (할 일 목록)

#### ✅ 좋은 점

1. **명확한 정보 구조**
   - Task 카드에 제목, 진행률 바, 진행률 텍스트가 모두 표시
   - 진행률을 시각(바)과 텍스트(3/5 완료)로 이중 표현 → 접근성 우수

2. **효율적인 레이아웃**
   - FlatList로 스크롤 성능 보장
   - FAB (+버튼) 위치가 엄지손가락 도달 가능 영역

3. **Empty State 제공**
   - "할 일을 추가해보세요!" 메시지로 사용자 가이드
   - 이모지 (📝) 추가로 친근함 표현

#### ⚠️ 개선 필요

1. **진행률 바 시각적 위계 (High)**
   - 문제: 진행률 바 높이가 8pt로 너무 얇을 수 있음
   - 제안: 10-12pt로 증가하여 시각적 강조
   - 이유: 앱의 핵심 가치인 "진행 가시화"를 더 강조

2. **Card 시각적 깊이 (Medium)**
   - 문제: elevation 2-3으로 명시되었으나 구체적인 그림자 값 없음
   - 제안:
     ```typescript
     shadow: {
       shadowColor: '#000',
       shadowOffset: { width: 0, height: 2 },
       shadowOpacity: 0.1,
       shadowRadius: 4,
       elevation: 3,
     }
     ```

3. **완료된 Task 시각적 차별화 (Medium)**
   - 문제: 100% 완료된 Task의 처리 방안 미정의
   - 제안 1: 완료 Task는 하단으로 이동 (선택적 정렬)
   - 제안 2: 완료 Task에 체크 아이콘 또는 배지 추가
   - 제안 3: 완료 Task를 접을 수 있는 토글 제공

4. **Task 개수가 많을 때 처리 (Low)**
   - 문제: Task 500개 시 스크롤 길이가 매우 김
   - 제안: 검색/필터 기능 (Phase 2) 또는 페이지네이션 고려

5. **FAB 그림자 명확화 (High)**
   - 문제: FAB는 화면 위에 떠 있는 버튼인데 그림자 미정의
   - 제안:
     ```typescript
     fabShadow: {
       shadowColor: '#3b82f6',
       shadowOffset: { width: 0, height: 4 },
       shadowOpacity: 0.3,
       shadowRadius: 8,
       elevation: 6,
     }
     ```

#### ✅ 좋은 점

1. **편집 가능한 제목**
   - 인라인 편집으로 빠른 수정 가능
   - 사용자 플로우가 간결

2. **진행률 섹션 강조**
   - 별도 Card로 진행률 표시 → 시각적 강조
   - 중앙 정렬로 집중도 향상

3. **Checklist 항목 구조**
   - 체크박스, 제목, 삭제 버튼이 명확
   - 완료 항목에 취소선 → 시각적 피드백

4. **Empty State**
   - "아직 세부 단계가 없습니다. 추가해보세요!" 메시지
   - 사용자 액션 유도

#### ⚠️ 개선 필요

1. **헤더 버튼 접근성 (Critical)**
   - 문제: 뒤로가기(←)와 삭제(🗑) 버튼의 터치 영역 미정의
   - 제안: 최소 44x44pt 보장, 아이콘 크기는 24x24pt
   - 추가: accessibilityLabel 명시 ("뒤로가기", "할 일 삭제")

2. **제목 편집 모드 명확화 (High)**
   - 문제: 제목을 탭하면 편집 모드가 된다고 했으나, 시각적 피드백 미정의
   - 제안:
     - 편집 모드: border 추가 (`#3b82f6`), 커서 표시
     - 뷰 모드: border 없음, 탭 가능 힌트 (연한 배경?)
   - 대안: 제목 옆에 편집 아이콘(✏️) 추가 (명확성 향상)

3. **Checklist 항목 편집 (High)**
   - 문제: 항목 제목을 탭하면 편집된다고 했으나, 체크박스 탭과 구분 어려움
   - 제안:
     - 체크박스: 좌측 44x44pt 영역
     - 제목 텍스트: 중앙 영역 (탭 시 편집)
     - 삭제 버튼: 우측 44x44pt 영역
   - 우려: 실수로 편집 모드 진입 가능성 → 제목 롱프레스로 편집 고려

4. **진행률 바 위치 및 크기 (Medium)**
   - 문제: 진행률 섹션이 Card로 되어있는데 여백이 명확하지 않음
   - 제안:
     - Card 패딩: 16px
     - 진행률 바 높이: 12px (TaskListScreen보다 크게)
     - 진행률 텍스트: 16px, center 정렬

5. **Checklist 순서 변경 기능 (Low)**
   - 문제: 항목 순서를 바꿀 방법 없음
   - 제안: 드래그 앤 드롭 (Phase 2) 또는 상하 화살표 버튼

6. **항목 삭제 확인 다이얼로그 (Medium)**
   - 문제: 삭제 확인 다이얼로그 텍스트 미정의
   - 제안:
     ```
     제목: "항목 삭제"
     메시지: "'{항목 제목}'을(를) 삭제하시겠습니까?"
     버튼: ["취소", "삭제"]
     ```
   - 삭제 버튼 색상: Danger Red

7. **완료 항목과 미완료 항목 구분 (Medium)**
   - 현재: 취소선으로만 구분
   - 제안: 완료 항목을 하단으로 이동 (선택적)
   - 또는: "완료된 항목 숨기기" 토글 제공

8. **키보드 처리 (High)**
   - 문제: 입력창 포커스 시 키보드로 인한 레이아웃 가림

### Modal (Task 추가)

#### ✅ 좋은 점

1. **간단한 구조**
   - 제목 입력과 버튼만 있어 집중도 높음
   - 닫기 버튼(✕)으로 취소 가능

2. **명확한 액션**
   - "취소"와 "추가" 버튼으로 의도 명확

#### ⚠️ 개선 필요

1. **모달 크기 및 위치 (Medium)**
   - 문제: 모달 크기가 와이어프레임에만 표시되고 구체적 수치 없음
   - 제안:
     - 화면 하단에서 올라오는 Bottom Sheet 스타일
     - 높이: 내용에 맞춰 자동 (최소 200pt)
     - 좌우 여백: 0 (전체 너비)
     - 모서리: 상단만 border-radius 16px

2. **TextInput 스타일 (High)**
   - 문제: 입력창 스타일 미정의
   - 제안:
     - placeholder: "할 일을 입력하세요" (16px, Text Secondary)
     - border: 1px solid #e5e7eb
     - focus border: 2px solid #3b82f6
     - padding: 12px
     - border-radius: 8px
     - multiline: true (여러 줄 지원)
     - maxLength: 120

3. **버튼 레이아웃 (Medium)**
   - 문제: 취소/추가 버튼 간격 및 크기 미정의
   - 제안:
     - 버튼 간격: 12px
     - 취소 버튼: 보조 버튼 스타일 (outline, 회색)
     - 추가 버튼: Primary 버튼 스타일 (filled, 파란색)
     - 버튼 높이: 48px (터치 영역 충분)

4. **배경 터치로 닫기 (Medium)**
   - 문제: 반투명 배경을 탭하면 모달이 닫히는지 미정의
   - 제안: 배경 탭 시 모달 닫기 (일반적인 UX 패턴)

5. **입력 검증 피드백 (High)**
   - 문제: 빈 제목 또는 120자 초과 시 처리 미정의
   - 제안:
     - 빈 제목: "추가" 버튼 비활성화 (opacity 0.6, 터치 불가)
     - 120자 초과: 입력창 하단에 "120/120" 글자 수 표시 (빨간색)

## 🔄 우선순위별 수정 사항

### Critical (즉시 수정 필요)

#### 1. 색상 대비 개선
- **문제**: Success Green, Danger Red 색상이 WCAG AA 미달
- **영향**: 시각 장애 사용자, 노인 사용자 등이 텍스트 읽기 어려움
- **해결**:
  - Success Green: `#10b981` → `#059669`
  - Danger Red: `#ef4444` → `#dc2626`
  - Text Secondary: `#6b7280` → `#4b5563`
- **작업량**: 5분 (색상 코드 변경)

#### 2. 터치 영역 명시
- **문제**: 체크박스, 삭제 버튼 등 터치 영역 미정의
- **영향**: 터치 실패, 사용성 저하, 손가락 큰 사용자 불편
- **해결**: 모든 인터랙티브 요소 최소 44x44pt 보장
- **작업량**: 10분 (디자인 토큰 추가)

#### 3. accessibilityLabel 추가
- **문제**: 스크린 리더 사용자가 앱 사용 불가
- **영향**: 시각 장애 사용자 접근 불가
- **해결**: 모든 버튼, 입력창, 카드에 레이블 추가
- **작업량**: 30분 (각 컴포넌트에 추가)

---

### High (출시 전 필수)

#### 4. 타이포그래피 줄 간격 정의
- **문제**: 줄 간격 없으면 텍스트 답답하고 가독성 저하
- **영향**: 사용자 경험, 가독성
- **해결**: 각 폰트 크기별 lineHeight 정의
- **작업량**: 5분

#### 5. 입력 검증 및 피드백
- **문제**: 빈 제목, 120자 초과 시 처리 미정의
- **영향**: 에러 발생, 혼란
- **해결**:
  - 빈 제목 시 버튼 비활성화
  - 120자 초과 시 경고 메시지
  - 글자 수 표시
- **작업량**: 20분

#### 6. 진행률 바 크기 증가
- **문제**: 8pt 높이는 너무 얇아 시인성 낮음
- **영향**: 앱의 핵심 가치 약화
- **해결**: 10-12pt로 증가
- **작업량**: 2분

#### 7. Card 그림자 정의
- **문제**: elevation 2-3만 명시, 구체적 값 없음
- **영향**: iOS/Android 플랫폼 간 일관성 없음
- **해결**: shadowColor, shadowOffset 등 명시
- **작업량**: 5분

#### 8. 제목 편집 모드 시각적 피드백
- **문제**: 편집 모드인지 불명확
- **영향**: 사용자 혼란
- **해결**: 편집 모드 시 border 추가, 아이콘 변경
- **작업량**: 15분

#### 9. 키보드 처리
- **문제**: 키보드로 인한 레이아웃 가림
- **영향**: 입력 중 내용 안 보임
- **해결**: KeyboardAvoidingView + 자동 스크롤
- **작업량**: 10분 (✅ 이미 일부 구현됨)

---
