# Split TODO - 앱 기획서

## 📱 앱 개요

### 앱 이름
**Split TODO** (스플릿 투두)

### 한 줄 설명
할 일을 작은 체크리스트로 쪼개어 진행도를 시각적으로 확인하는 TODO 앱

### 핵심 가치 제안
- **세분화**: 큰 할 일을 작은 단계로 나누어 부담을 줄임
- **진행 가시화**: 실시간 진행률로 성취감 제공
- **심플함**: 복잡한 기능 없이 필수 기능만 제공
- **오프라인**: 인터넷 없이도 완벽하게 작동

---

## 🎯 타겟 사용자

### 주요 타겟
- **학생**: 과제, 프로젝트를 단계별로 관리
- **직장인**: 업무를 세분화하여 체계적으로 처리
- **프리랜서**: 프로젝트를 작은 단계로 나누어 관리
- **목표 달성자**: 장기 목표를 단계별로 실행

### 사용자 페르소나

**페르소나 1: 대학생 김민수 (22세)**
- 과제와 프로젝트가 많아 압도됨
- 할 일을 시작하기 전에 막막함
- 작은 성취감이 필요함
- 사용 시나리오: "졸업 논문 작성"을 10개의 작은 단계로 나누어 하루에 1-2개씩 완료

**페르소나 2: 프로덕트 매니저 이지은 (29세)**
- 복잡한 기능이 많은 앱은 오히려 비효율적
- 빠르게 할 일을 기록하고 진행도를 확인하고 싶음
- 사용 시나리오: "신규 기능 출시"를 기획, 디자인, 개발, 테스트 단계로 나누어 팀원들과 공유

**페르소나 3: 프리랜서 박준호 (35세)**
- 여러 프로젝트를 동시에 관리
- 각 프로젝트의 진행 상황을 한눈에 파악하고 싶음
- 사용 시나리오: 3개의 클라이언트 프로젝트를 각각 체크리스트로 관리하며 우선순위 결정

## 💡 핵심 기능

### 1. Task 관리
**기능 설명**: 큰 할 일(Task)을 생성하고 관리

**사용자 흐름**:
1. + 버튼 클릭
2. Task 제목 입력 (예: "웹사이트 리뉴얼")
3. 생성 → 목록에 표시

**화면**: TaskListScreen

### 2. Checklist 세분화
**기능 설명**: Task를 작은 단계(Checklist)로 나누기

**사용자 흐름**:
1. Task 카드 클릭
2. 상세 화면 진입
3. "세부 단계 추가" 입력창에 항목 입력
   - 예: "디자인 시안 작성"
   - 예: "개발 환경 설정"
   - 예: "프론트엔드 구현"
4. + 버튼으로 추가

**화면**: TaskDetailScreen

### 3. 진행률 시각화
**기능 설명**: 완료된 항목 수 기반 진행률 표시

**표시 형식**:
- 진행률 바 (0-100%)
- 텍스트: "3/5 완료 (60%)"
- 색상: 진행 중(파란색), 완료(초록색)

**계산 방식**:
```
진행률(%) = (완료된 항목 수 / 전체 항목 수) × 100
```

### 4. 체크/해제
**기능 설명**: 완료한 항목을 체크하고 해제

**사용자 흐름**:
1. 체크박스 탭
2. 즉시 체크 상태 전환 (100ms 이내)
3. 진행률 자동 업데이트
4. 완료 항목은 취소선 표시

### 5. 편집/삭제
**기능 설명**: Task와 Checklist 항목 수정 및 삭제

**편집**:
- Task 제목: 탭하여 인라인 편집
- Checklist 항목: 탭하여 인라인 편집

**삭제**:
- Task: 🗑 아이콘 → 확인 다이얼로그
- Checklist 항목: ✕ 버튼 → 확인 다이얼로그

### 6. 오프라인 저장
**기능 설명**: 모든 데이터를 로컬에 자동 저장

**저장 방식**:
- AsyncStorage 사용
- 500ms debounce (빠른 연속 변경을 1회로 저장)
- 자동 저장 + 백업 저장
- 실패 시 1회 재시도

## 📊 데이터 구조

### 데이터 모델

```typescript
// 체크리스트 항목
interface ChecklistItem {
  id: string;           // UUID v4
  title: string;        // 1~120자
  done: boolean;        // 완료 여부
}

// 할 일 (Task)
interface Task {
  id: string;           // UUID v4
  title: string;        // 1~120자
  items: ChecklistItem[]; // 체크리스트 (최대 200개)
  createdAt: string;    // ISO 8601
  updatedAt: string;    // ISO 8601
  schemaVersion?: number; // 기본 1
}

// 전체 앱 데이터
interface AppData {
  schemaVersion: number; // 기본 1
  tasks: Task[];        // Task 목록 (최대 1000개)
}

// 진행률
interface Progress {
  done: number;         // 완료된 항목 수
  total: number;        // 전체 항목 수
  percent: number;      // 퍼센트 (0-100)
}
```

### 데이터 저장

**Storage Keys**:
- `APP_DATA`: 메인 데이터
- `APP_DATA_BACKUP`: 백업 데이터

**저장 타이밍**:
- 데이터 변경 후 500ms debounce
- 앱 종료 시 자동 저장
- 백그라운드 전환 시 자동 저장

**데이터 크기 제한**:
- Task: 최대 1000개 (초과 시 경고)
- Checklist 항목: Task당 최대 200개

## 📱 비기능 요구사항

### 성능

**로딩 시간**:
- 초기 로딩: 500ms 이내
- 화면 전환: 100ms 이내

**UI 반응성**:
- 체크박스 탭: 100ms 이내 반영
- 입력 반응: 즉시 (debounce 없음)
- 저장: 500ms debounce (백그라운드)

**대량 데이터**:
- Task 500개 기준 테스트
- Checklist 항목 100개 기준 테스트
- 원활한 스크롤 보장

### 안정성

**데이터 보호**:
- 자동 저장 + 백업 저장
- 저장 실패 시 1회 재시도
- 스키마 마이그레이션 지원

**에러 처리**:
- 저장 실패: 사용자에게 알림
- 네트워크 불필요: 오프라인 완전 지원
- 앱 크래시 방지: try-catch

### 접근성 (A11y)

**스크린 리더 지원**:
- 모든 버튼에 accessibilityLabel
- 체크박스: "항목명, 완료됨/미완료"
- 진행률: "5개 중 3개 완료, 60퍼센트"

**시각적 접근성**:
- 색상 대비 WCAG AA 이상 (4.5:1)
- 진행률을 색상과 텍스트로 표시
- 충분한 터치 영역 (최소 44x44)

**키보드 네비게이션**:
- Tab 순서: 논리적 흐름
- Enter로 입력 제출
- Esc로 모달 닫기

### 플랫폼 지원

**iOS**:
- iOS 13.0 이상
- iPhone, iPad 지원
- Safe Area 대응

**Android**:
- Android 5.0 (API 21) 이상
- 다양한 화면 크기 대응
- Material Design 가이드 준수

## 🎯 차별화 포인트

### 경쟁 앱 vs Split TODO

| 기능 | Todoist | Notion | Split TODO |
|------|---------|--------|------------|
| 세분화 중심 | ❌ | ⚠️ (복잡) | ✅ |
| 진행률 표시 | ⚠️ (기본) | ❌ | ✅ (실시간) |
| 심플함 | ⚠️ | ❌ | ✅ |
| 오프라인 | ✅ | ⚠️ | ✅ |
| 무료 | ⚠️ (제한) | ⚠️ (제한) | ✅ |

### 우리만의 강점
1. **세분화에 특화**: 체크리스트가 핵심 기능
2. **진행 가시화**: 실시간 진행률로 동기부여
3. **초심플**: 30초 안에 사용법 습득
4. **완전 무료**: 모든 기능 무료 제공
5. **100% 오프라인**: 네트워크 불필요

## 📝 결론

**Split TODO**는 할 일을 작은 단계로 나누어 관리하는 데 특화된 심플한 TODO 앱입니다.

**핵심 철학**:
> "큰 목표는 작은 단계로 나누면 달성 가능하다"

**목표 사용자 경험**:
> "처음 사용해도 30초 안에 할 일을 추가하고 세분화할 수 있다"
